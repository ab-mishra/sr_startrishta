Progress Bars

Quarca uses a jQuery plugin called Bootstrap-Progressbar to extend the normal Bootstrap progress bars. Check out the link for more information.


How to use progress bars?

Step 1: Add the jQuery plugin after the wrapper div. <script src="vendor/plugins/ui/progress-bar/bootstrap-progressbar.min.js"></script>
Step 2: Add the HTML code.
Step 3: Initialize the plugin before the ending body tag.

$('.progress .progress-bar').progressbar({

    transition_delay: 400,

    display_text: 'fill'

});

How to use different colors?

Add progress-bar-success, progress-bar-info, progress-bar-warning or progress-bar-danger in the second div after progress-bar.


How to initialize the progress bar from the right?

Add right in the first div after the class name progress.


How to use animated progress bars?

Add progress-bar-striped active in the second div class after progress-bar.


How to use vertical progress bars?

Add vertical in the first div after progress.


Modals

IMPORTANT:

Make sure to include the modal HTML between </div><!-- main --> and </div><!-- wrapper --> ending tags for the modal to open without causing any bugs.


Why can't I include it right after the modal trigger button?

Because of how the Sidebar/Content toggle(open/close sidebar) works. You will notice that when the sidebar is toggled, the sidebar stays, only the content part moves. So, the sidebar stays; the header is fixed; the content moves to hide the sidebar but also goes under the header when scrolling.

In order to achieve this:
1. The header has to be in a fixed position for the content to scroll under it. It requires a higher z-index value to be over the content part.
2. For the content part to hide the sidebar, it requires a higher z-index value than the sidebar. But, the z-index value of the content part can't be higher than the header or it will be over the header when scrolling.
3. Now, when you add a modal into the content, when it opens it has a fixed position. As the content part has a relative position(to be able to add the z-index), the header will still be visible when opening the modal and will cause some bugs.

The solution is simple, move the modal outside of the content part(after main ending tag).

How to use different colors?

Add modal-primary, modal-success, modal-warning, modal-danger or modal-dark after modal in the first div class.


How to use left/right/bottom/full-screen modals?

Add left, right, bottom or full after modal in the first div class.


How to mix styles?

By simply adding the color and position classes on the same class after modal. It will work!

Carousel

Quarca uses this great plugin Owl Carousel 2 for fully responsive and touch enabled carousels. It has over 60 options and works on all modern browsers. You are not limited to the demo in Quarca, it is pretty easy to use the owl options and extend the functionalities, check out the website for more information.


How to use the plugin?

Step 1: Add the CSS inside of the head tag. <link href="vendor/plugins/ui/carousel/owl.carousel.css" rel="stylesheet">
Step 2: Add the JS after the wrapper div <script src="vendor/plugins/ui/carousel/owl.carousel.min.js"></script>
Step 3: Add the HTML and initialize the plugin.
